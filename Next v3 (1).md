# **Appendix E — Friction Engine Specification**

## **Overview**

The **Friction Engine** is the core behavioral system in Next v3.

Its purpose is to:

**minimize the psychological and cognitive effort required to start a task**

It transforms large or resisted tasks into the **smallest possible actionable step** that the user is likely to begin.

---

## **E1. Core Principle**

The Friction Engine operates on one fundamental rule:

**If a user is not starting, the task is too big.**

The system must continuously reduce task difficulty until:

* the user starts

* or the task becomes trivially easy

---

## **E2. Definitions**

### **Friction**

Friction represents the **mental resistance to starting a task**.

It is influenced by:

* perceived effort

* uncertainty

* emotional resistance

* fatigue

---

### **Activation Energy**

The minimum effort required for a user to begin an action.

Goal:

`Activation Energy → Near Zero`

---

## **E3. Inputs**

The Friction Engine uses the following inputs:

### **1\. Task Complexity**

* estimated time

* number of steps

* ambiguity level

---

### **2\. User Resistance Signals**

Detected via:

* task skips

* “not now” responses

* delayed starts

* repeated postponement

---

### **3\. Energy Level**

From Energy Model:

`LOW / MEDIUM / HIGH`

---

### **4\. Historical Behavior**

* past task completion rates

* previous resistance patterns

* preferred task sizes

---

## **E4. Output**

The engine outputs:

`NextAction`

Where:

* action is immediately executable

* requires minimal thinking

* has clear start condition

---

## **E5. Task Scaling Levels**

Tasks are scaled across levels.

| Level | Description | Example |
| ----- | ----- | ----- |
| L0 | Full task | Write essay |
| L1 | Reduced task | Write introduction |
| L2 | Small task | Write one paragraph |
| L3 | Micro task | Write one sentence |
| L4 | Start task | Open document |

---

## **E6. Scaling Algorithm**

### **Step 1 — Evaluate Resistance**

`if resistance == NONE:`  
    `use L1 or L2`  
`elif resistance == LOW:`  
    `use L2 or L3`  
`elif resistance == HIGH:`  
    `use L3 or L4`

---

### **Step 2 — Adjust for Energy**

`if energy == LOW:`  
    `shift down one level`  
`if energy == HIGH:`  
    `allow larger tasks`

---

### **Step 3 — Ensure Clarity**

Every task must:

* start with an action verb

* have a clear first step

* require no planning

---

## **E7. Start-Only Mode (Critical Feature)**

When resistance is very high, the system switches to:

`START-ONLY MODE`

Tasks in this mode:

* require \< 2 minutes

* focus only on beginning

* do not require completion

Example:

`"Open your notes"`  
`"Write one word"`  
`"Read one line"`

---

## **E8. Success Criteria**

A task is considered successful if:

* user begins action

* not necessarily completes it

This is critical.

---

## **E9. Feedback Loop**

After each task:

`User completes task`  
 `↓`  
`Reduce resistance score`  
 `↓`  
`Increase next task size slightly`

If user fails:

`Increase resistance score`  
 `↓`  
`Shrink next task`

---

## **E10. Anti-Failure System**

If a user repeatedly fails:

1. reduce task size aggressively

2. switch to start-only mode

3. optionally suggest break

---

## **E11. Task Generation Constraints**

All tasks must:

* be physically actionable

* be specific

* avoid abstract language

---

### **BAD Task**

`"Work on your project"`

---

### **GOOD Task**

`"Open your project file"`

---

## **E12. Cognitive Load Minimization**

Tasks must avoid:

* multiple steps

* decision-making

* ambiguity

Each task \= **one action only**

---

## **E13. Time Constraint Enforcement**

Max task time:

`≤ 40 minutes`

Minimum:

`≤ 2 minutes (start mode)`

---

## **E14. Progressive Expansion**

As momentum builds:

`L4 → L3 → L2 → L1`

System gradually increases difficulty.

---

## **E15. Behavioral Reinforcement**

After action:

Sage responds with:

* acknowledgment

* not praise overload

Example:

`"Good. Let’s continue."`

---

## **E16. Edge Cases**

### **Case 1 — User refuses all tasks**

Action:

`Switch to break suggestion`

---

### **Case 2 — User overwhelmed**

Action:

`Offer reset task:`  
`"Take one deep breath"`

---

### **Case 3 — User disengaged**

Action:

`Passive nudge later`

---

## **E17. System Constraints**

The Friction Engine must:

* never increase task difficulty after failure

* never present multiple tasks

* never require planning from the user

---
